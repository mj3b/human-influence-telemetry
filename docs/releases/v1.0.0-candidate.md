# Human Influence Telemetry 1.0.0 Candidate Release Outline

**Status:** Candidate outline, release prohibited  
**Current repository release:** `0.6.0`  
**Current normative contract:** `0.4.0`  
**Current conformance engine:** `0.5.0`  
**Current maturity:** Level 2, Applicable

## Intended release meaning

Version `1.0.0` will declare that the public HIT assessment contract is stable enough for independent implementation and compatibility commitments.

It will not declare population-wide human reliability, causal validity, legal conformity, certification, or independent institutional adoption.

## Planned stable components

- specification `1.0.0`;
- assessment schema `1.0.0`;
- dimension catalog `1.0.0`;
- application handbook `1.0.0`;
- conformance engine `1.0.0`;
- compatibility manifest and migration guide;
- implementation packet;
- clean-room audit record;
- current public applications or documented migration exceptions;
- claim, limitation, provenance, citation, and release metadata.

## Completed foundation

The candidate inherits the completed `0.4.0`, `0.5.0`, and `0.6.0` work:

- stable normative rules for the six dimensions and split Telemetry Integrity;
- explicit evidence-state and finding invariants;
- actor, authority, Repair, sampling, aggregation, contradiction, and locator rules;
- 48 deterministic boundary fixtures;
- complete-assessment conformance and stable error codes;
- public CLI and deterministic reports;
- first bounded independent human result and Maturity Level 2;
- protected historical-version and migration controls.

## Current blockers

Release `1.0.0` remains prohibited until:

1. the current-contract packet and application path is resolved;
2. at least three current-contract public applications exist, or case-specific migration exceptions are documented;
3. a standalone implementation packet is complete;
4. a clean-room implementation audit is complete;
5. a public `v0.9.0` release candidate is published;
6. release-blocking audit findings are repaired;
7. component versions are promoted and synchronized;
8. the breaking-change review passes;
9. exact-release validation passes.

## Human replication boundary

The planned three-scorer replication is the preferred empirical path. It is not silently converted into a lower standard when recruitment is difficult.

A complete replication may advance H3 and research maturity. Incomplete recruitment must remain reported as incomplete recruitment. The semantic stability claim for `1.0.0` remains separate from the empirical maturity claim.

## Manual workbook readiness

Draft manual workbooks exist for three scorers and three future packets. They are 31-page candidate assets in DOCX and PDF form. They remain visibly marked `DRAFT - SCORING PROHIBITED` until `v0.7.0` locks packet identifiers and permits scoring.

## Mandatory non-claims

The release must state that HIT is a documentary assurance method. It does not perform runtime governance, signed-receipt verification, policy-pack harmonization, compliance automation, or evidence portability.

The release must also state the exact H3 and maturity evidence available at publication time.

## Publication control

This file is an outline only. It does not authorize version promotion, tag creation, release publication, DOI archival, or activation of the scorer workbooks.
