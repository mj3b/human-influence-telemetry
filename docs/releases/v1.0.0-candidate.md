# Human Influence Telemetry 1.0.0 Candidate Release Outline

**Status:** Candidate outline, release prohibited  
**Current public repository release:** `0.6.0`  
**Stable-release target:** `1.0.0`

## Intended release meaning

Version `1.0.0` will declare that the public HIT assessment contract is stable enough for independent implementation and compatibility commitments.

It will not declare population-wide human reliability, causal validity, legal conformity, certification, or independent institutional adoption.

## Current blockers

Release `1.0.0` remains prohibited until:

1. the human case-selection decision is signed;
2. three current-contract packets are frozen with exact boundaries, source manifests, stable identifiers, and digests;
3. the `v0.7.0` protocol package is locked and published;
4. the current-contract public-application path is resolved;
5. at least three current-contract public applications exist, or case-specific migration exceptions are documented;
6. a standalone implementation packet is complete;
7. a clean-room implementation audit is complete;
8. a public `v0.9.0` release candidate is published;
9. release-blocking audit findings are repaired;
10. component versions are promoted and synchronized;
11. the breaking-change review passes;
12. exact-release validation passes.

## Human replication boundary

The planned three-scorer replication is the preferred empirical path. Recruitment difficulty does not authorize a silent reduction in the locked standard.

A complete replication may advance H3 and research maturity. Incomplete recruitment must remain reported as incomplete recruitment. The semantic stability claim for `1.0.0` remains separate from the empirical maturity claim.

## Manual workbook readiness

Draft manual workbooks exist for three scorers and three future packets. They are 31-page candidate assets in DOCX and PDF form. They remain visibly marked `DRAFT - SCORING PROHIBITED` until `v0.7.0` locks packet identifiers and permits scoring.

## Clean-room boundary

The candidate implementation packet tests whether an eligible external reviewer can install, operate, and explain HIT using public materials without private author explanation. Audit activation remains prohibited until the exact repository commit, environment, expected outputs, hashes, submission schema, and auditor eligibility record are frozen.

A failed clean-room audit remains evidence. Release-blocking defects must be published and repaired before `v0.9.0` promotion or `v1.0.0` release.

## Mandatory non-claims

The release must state that HIT is a documentary assurance method. It does not perform runtime governance, signed-receipt verification, policy-pack harmonization, compliance automation, or evidence portability.

The release must also state the exact H3 and maturity evidence available at publication time.

## Future GitHub release record

The GitHub release title, tag, body, citation version, component versions, and asset manifest must be generated from the exact validated release commit. Candidate wording from this file may be reused only after every blocker above is closed.

The release body must include:

- the stable-contract meaning of `1.0.0`;
- the exact public component versions;
- the research maturity level and H3 status at release time;
- migration and compatibility boundaries;
- clean-room audit disposition;
- public application count and provenance;
- mandatory non-claims;
- release-asset names, sizes, and SHA-256 hashes;
- DOI status based on completed archival, with no provisional identifier presented as final.
